# Řešení problémů

## Server se nespustí

### `ModuleNotFoundError: No module named 'czechmedmcp'`

```bash
# Přeinstalujte balíček
uv pip install -e .
# Nebo
pip install czechmedmcp
```

### `command not found: czechmedmcp`

```bash
# Ověřte instalaci
uv tool list | grep czechmedmcp
# Nebo zkuste přes uv run
uv run czechmedmcp run
```

## Claude Desktop nevidí nástroje

1. Ověřte konfiguraci v `claude_desktop_config.json`
2. Restartujte Claude Desktop
3. Zkontrolujte, že `uv` je v PATH:
   ```bash
   which uv
   ```
4. Vyzkoušejte absolutní cestu v konfiguraci:
   ```json
   {
     "mcpServers": {
       "czechmedmcp": {
         "command": "/Users/vase_jmeno/.cargo/bin/uv",
         "args": ["run", "--with", "czechmedmcp", "czechmedmcp", "run"]
       }
     }
   }
   ```

## Pomalé odpovědi

CzechMedMCP cachuje HTTP odpovědi. První dotaz může být pomalší, následné budou rychlejší.

- Cache TTL: 1 týden (výchozí)
- Umístění: `~/.cache/czechmedmcp/http_cache`
- Vymazání: `rm -rf ~/.cache/czechmedmcp/`

## Chyby českých zdrojů

### SUKL API neodpovídá

SUKL API může být občas pomalé nebo nedostupné. CzechMedMCP má vestavěný retry s exponenciálním backoffem a circuit breaker.

Timeout: 30 sekund (konfigurovatelný přes `CZECH_HTTP_TIMEOUT`).

### MKN-10 data se nenačítají

MKN-10 data se načítají do paměti při prvním dotazu (~20 MB). Pokud načtení selže, zkuste:

```bash
# Ověřte konektivitu
curl -s https://mkn10.uzis.cz/api/ | head -c 100
```
