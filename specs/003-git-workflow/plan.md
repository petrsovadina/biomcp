## Implementation Phases

### Phase 1: Branch konsolidace (US1 — P1)

**Cíl**: python-main obsah → main, push na remote

**Kroky**:
1. Checkout `main`, reset na `python-main` HEAD
2. Force-push `main` na `origin/main`
3. Smazat lokální `python-main`
4. Nastavit tracking `main` → `origin/main`

**Verifikace**: `git log --oneline -5 main` ukazuje rename commit, `git status` ukazuje up-to-date s remote

### Phase 2: Cleanup stale references (US2 — P1)

**Cíl**: Odstranit všechny stale tracking references

**Kroky**:
1. `git fetch --prune` pro cleanup remote refs
2. Smazat stale lokální branches (pokud existují)
3. Odstranit stale tracking config z `.git/config`
4. Ověřit `git branch -vv` — pouze `main` s platným tracking

**Verifikace**: `git branch -vv` ukazuje pouze `* main → origin/main`

### Phase 3: CI aktualizace (US4 — P2)

**Cíl**: CI workflow targetuje `main` místo `python-main`

**Kroky**:
1. Upravit `.github/workflows/ci.yml` — branch triggers
2. Commitnout změnu na feature branch
3. Po merge ověřit, že CI se spustí

**Verifikace**: Push na main spustí CI

### Phase 4: Dokumentace branching modelu (US3 — P2)

**Cíl**: CLAUDE.md obsahuje sekci "Git Workflow"

**Kroky**:
1. Přidat sekci do CLAUDE.md s pravidly:
   - Hlavní branch: `main`
   - Feature branches: `NNN-feature-name` (speckit)
   - Workflow: speckit.specify → speckit.plan → speckit.tasks → speckit.implement
   - Merge: PR do `main`, po merge smazat branch
   - Commit style: conventional commits
2. Aktualizovat constitution references (python-main → main)

**Verifikace**: CLAUDE.md obsahuje sekci s min. 10 řádky pravidel

### Phase 5: Finální verifikace (všechny SC)

**Kroky**:
1. SC-001: `git branch -a` — pouze `main` + `remotes/origin/main`
2. SC-002: `git branch -vv` — `main` s platným tracking
3. SC-003: `git status` — clean, up-to-date
4. SC-004: Push trigger test nebo `gh workflow view ci.yml`
5. SC-005: `grep -c "Git Workflow" CLAUDE.md` > 0
6. SC-006: Test speckit `create-new-feature.sh` — ověřit formát
